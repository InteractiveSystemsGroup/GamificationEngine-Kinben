RoleApi Resource

A role describes which members of an organisation is allowed to do or see particular elements of the engine such as to fulfil a particular task and get its rewards. Each Player can have many different roles such as one for his occupation or the department in which she/he works. But the roles can also be a part of an invented role system that isnâ€™t oriented towards the work context. All roles are specific to the respective created organisation. Ancillary creating and deleting, either all roles of a specific organisation or with a given id the associated role can be gotten. The name of one role can also be changed at a later point of time.

POST /role

Creates a new role for the organisation to which the API key belongs to. Because of this API key the created role is specific to this organisation. If the API key is not valid an analogous message is returned.

Request Parameters
name type description
apiKey query The valid query parameter API key affiliated to one specific organisation, to which this role belongs to.
roleName query The required name of the new role. This field must not be null.
Response Body
media type data type description
application/json Role (JSON) Response of Role in JSON.

GET /role/{id}

Gets a specific role of an organisation so the id of the organisation and its API key are needed. If the API key is not valid an analogous message is returned. It is also checked, if the id is a positive number otherwise a message for an invalid number is returned.

Request Parameters
name type description
id path Required path parameter id of the role that should be gotten. This parameter is required.
apiKey query The valid query parameter API key affiliated to one specific organisation, to which this role belongs to.

media type data type description

PUT /role/{id}/attributes

With this method the name field of one role can be changed. For this the id of the role, the API key of the specific organisation, the name of the field and the new value are needed. If the API key is not valid an analogous message is returned.

Request Parameters
name type description
id path The id of the role that should be changed. This parameter is required.
apiKey query The valid query parameter API key affiliated to one specific organisation, to which this role belongs to.
attribute query The name of the attribute which should be changed. This parameter is required. The following names of attributes can be used to change the associated field: "name".
value query The new value of the attribute. This parameter is required.
Response Body
media type data type description
application/json Role (JSON) Response of Role in JSON.
